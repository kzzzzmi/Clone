HTML의 목적
브라우저에게 content를 이해시켜주는 것

<tag>
 -ol(ordered list) : 순서가 있는 목록 (ex. 1 2 3 4 5)
 -ul(unordered list) : 순서가 없는 목록
 -li(list item) : 목록 요소
 -a(anchor) : 닻 == list (다른 웹사이트로 이동하는 방법)
 -img : self closing tag(text가 아님)

<attributes> : 태그에 부가적인 정보를 추가하는 것
 -href(Hyperlink reference, HTTP reference) : 이동할 곳을 알려줌
 -target=(초기값)"_self"(새로운 창)"_blank"
 -src

각 태그에 맞는 attributes가 있다.
만일 잘못된 태그나 올바른 태그의 잘못된 attributes를 사용한다면 브라우저는 이해하지 못한다


<!DOCTYPE html> : 브라우저에게 text파일이 아닌 html문서라고 알려줌
<html></html> : 사이의 코드는 html 코드가 되는 것

웹사이트 구조
head : 웹사이트의 환경을 설정해줌(페이지에 관한 환경설정)
 - 쉽게 생각하면 google같은 검색엔진들이 이해할 수 있게 정리해서 말해주는 것
body : 사용자가 볼 수 있는 content를 보여줌

meta tag of head
1. content : 검색엔진에서의 title 부가설명
2. name
3. charset : 브라우저에게 text를 어떻게 그려야하는지 알려줌(utf-8)

label tag는 input tag와 함께 쓰인다.

id type == unique identifier(고유 식별자)
태그 하나당 하나의 id만 가질 수 있음
*중복하면 안됨

semantic html (의미를 파악할 수 있음) : 딱히 실행하는 것은 없음(container)
div(division) : 박스라고 생각하면 됨
header : 제목부분이란걸 이해할 수 있음
main : 중심부
footer : 꼬릿말
장점 : 가독성, 봤을 때 구조를 이해하기 편함
